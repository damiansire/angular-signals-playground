§ 04 · El grafo

El diamante de
dependencias

Una fuente alimenta dos derivados, y un tercero los vuelve a juntar. El miedo razonable: que ese tercero se asome cuando un brazo ya cambió y el otro no — un valor a medio cocer, un glitch. Movés la fuente y el grafo te muestra que eso nunca pasa.

Tomemos n como fuente. De ella cuelgan dos computed: el cuadrado {{ square() }} y la mitad {{ half() }}. Un tercer computed, report, los suma. La pregunta del gotcha no es cuánto da, sino cuántas veces se calcula.

Con n = {{ n() }}, el informe vale {{ report() }}, y lo hace en una sola pasada consistente: Signals no evalúa report hasta que ambos brazos quedaron al día. No hay estado intermedio observable. Por eso el contador de evaluaciones avanzó a {{ shownEvals() }} — uno por cada cambio de la fuente, jamás dos. Esa es la garantía glitch-free.

{{ n() }}

El vértice es read-only: a report nunca le asignás un valor, lo derivás.

n      = signal({{ n() }});
square = computed(() => n() * n());
half   = computed(() => n() / 2);
report = computed(() => square() + half());

recálculos de report: {{ shownEvals() }}
